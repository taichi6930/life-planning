# Life Planning プロジェクト

## プロジェクト概要

老後生活を計算し、60歳までの貯蓄目標を設定するアプリケーション。
最初のMVPでは「Coast File」計算機能を実装。

- **主要機能**: 年金計画、iDeCo計画、Coast FI目標設定
- **対象フレームワーク**: Flutter （Web + iOS + Android で統一）
- **展開環境**: 
  - Web: Cloudflare (Flutter Web)
  - iOS: AppStore (Flutter iOS)
  - Android: Google Play (Flutter Android)

## 重要なドキュメント

### 開発プロセス
- **AI-DLC ワークフロー**: [.github/copilot-instructions.md](.github/copilot-instructions.md)
- **ブランチ戦略**: [docs/branch-strategy.md](docs/branch-strategy.md)
  - GitHub Flow採用
  - main (本番) ブランチのみ
  - feature/* (人間) + copilot/* (自動) ブランチ
  - コミットメッセージ: Conventional Commits形式
- **デプロイメント**: [docs/deployment-flow.md](docs/deployment-flow.md)
  - main マージ → Staging環境へ自動デプロイ
  - Tag作成 → Production環境へ自動デプロイ

### プロジェクト設定
- **セキュリティル**: security-baseline 拡張ルール有効
- **ブランチ命名**:
  - `feature/xxx` = 人間が作成するフィーチャーブランチ
  - `copilot/xxx` = Copilot が自動で作成するブランチ
  - どちらも PR → main にマージ
- **デプロイメント**: Cloudflare (Web), AppStore/Play (モバイル)

## 🚫 ブランチワークフロー強制ルール

**重要: 以下のルールを厳密に守ること**

### Rule 1: main ブランチへの直接コミット禁止
- ❌ `git commit` を main ブランチで実行してはいけない
- ✅ すべての変更は `feature/*` または `copilot/*` ブランチで実行
- 理由: 品質管理、レビュー体制、チームコラボレーション

### Rule 2: デリバリは PR 経由のみ
- すべての実質的な変更は Pull Request経由で main にマージ
- main へのマージ前に Code Review が必須
- PR がマージされると、Staging および Production へ自動デプロイ

### Rule 3: ブランチ命名規則
- `feature/coast-file-calc` = 人間が作成（Coast File計算機能など）
- `copilot/dark-mode-ui` = Copilot が作成（UIダークモード対応など）
- 説明的で簡潔なスコープを含めること

### Rule 4: コミットメッセージ形式
- Conventional Commits: `feat/fix/docs/style/refactor/perf/test/chore(scope): message`
- 例: `feat(coast-file): 年金計算エンジン実装`
- 例: `docs(deployment): デプロイフロー説明追加`

### Rule 5: PR Review 要件
- 最低 1 名以上の review 者による approval が必須
- Repository owner が approve を行うなど、権限フレームワークを構築すること

### 実行例
```bash
# ❌ 悪い例 (main で直接コミット)
git checkout main
git commit -m "docs: update deployment flow"

# ✅ 良い例 (feature branch で実施)
git checkout -b feature/update-deployment-flow
git commit -m "docs(deployment): update deployment flow guidance"
git push origin feature/update-deployment-flow
# → PR を create → Review → Merge
```
