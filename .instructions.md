# Life Planning プロジェクト - AI エージェント指示書

## 言語とドキュメント作成

- **すべてのドキュメント**: 日本語（日本語）で記述すること
- **コードコメント**: 必要に応じて英語も可
- **コミットメッセージ**: 日本語で記述、明確で説明的なものにすること
- **テスト説明**: 日本語で記述すること

## 図表とビジュアライゼーション

- **すべての図表**: Mermaid形式で作成
  - フローチャート: `flowchart` または `graph` を使用
  - クラス図: `classDiagram` を使用
  - シーケンス図: `sequenceDiagram` を使用
  - ガントチャート: `gantt` を使用
  - その他の図表: 適切な Mermaid 形式を使用
- **絶対に使用しない**: ASCII art や プレーンテキスト図表
- **必ずレンダリング**: 複雑な図表は視覚形式で表示すること

## バージョン管理戦略

### ブランチ規約
- **GitHub Flow** を採用
- **デフォルトブランチ**: `main` (本番環境対応)
- **開発ブランチ作成ルール**:
  - **Copilot ブランチ**: `copilot/xxx` 形式を使用
    - AI エージェントによる自動生成や支援を含むブランチ
    - 例: `copilot/add-project-instructions`, `copilot/implement-unit1`
  - **機能ブランチ**: `feature/xxx` 形式（手動開発用）
  - **修正ブランチ**: `fix/xxx` 形式（バグ修正用）
  - **ドキュメント**: `docs/xxx` 形式（ドキュメント更新用）
  - 1つのブランチ = 1つの機能・修正

### main ブランチへのマージ
- **直接プッシュは禁止**: すべての変更は PR 経由で行うこと
- **PR のワークフロー**: 
  1. feature/copilot/xxx ブランチを作成
  2. コミットしてプッシュ
  3. PR を作成（説明とコンテキストを記載）
  4. レビュー・承認後に main へマージ
- **コミットメッセージ**: 日本語で、変更内容を明確に（例：`feat: プロジェクト指示書を追加`）

### リモート設定
- **Remote**: `origin` (GitHub)
- **Repository**: https://github.com/taichi6930/life-planning

## プロジェクト構成

### 実装ユニット（4段階アプローチ）
1. **Unit 1**: Shared Business Logic（年金計算エンジン）
2. **Unit 2**: Web 実装（React/Vue + Cloudflare）
3. **Unit 3**: iOS 実装（Swift or React Native）
4. **Unit 4**: Android 実装（Kotlin or React Native）

### デプロイメント対象
- **Web**: Cloudflare Workers + D1 SQLite
- **モバイル**: App Store / Google Play

## セキュリティとコード品質

- **セキュリティベースライン**: security-baseline 拡張ルールに従うこと
- **本番環境グレード**: すべてのコードが個人情報保護要件を満たすこと
- **コンプライアンス**: 個人情報保護法（APPI）への対応必須

## ファイル組織

- `.gitignore`: 多言語プロジェクト対応（Node.js, Python, Ruby, Java など）
- `.github/`: プロジェクト設定（必要に応じて）
- `.instructions.md`: このファイル（AI エージェント指示）
- 各ユニット間の関心の分離を徹底

## ファイル作成時のチェックリスト

1. 正しいユニット・モジュール配下か確認
2. 日本語ドキュメント標準に従う
3. 図表は Mermaid のみで作成
4. セキュリティベースラインを確認
5. 日本語で明確なコミットメッセージを記載
6. PR 経由で main へマージする
7. PR マージ前に十分なテストを実施

## 重要な注記

- **データベース**: SQLite3（ポータブル、本番環境対応）
- **個人情報**: PII 保護要件に従い慎重に取り扱うこと
- **コードレビュー**: すべての PR はマージ前にレビューが必須
- **テスト**: コード実装と同時にテストを作成すること（説明は日本語）
